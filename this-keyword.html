🎯 this Keyword

this Keyword

How "this" changes based on how a function is called

"this" refers to the execution context of a function — who is calling it. In regular functions, this is determined at call time. In arrow functions, this is locked at definition time (lexical). Understanding this distinction is critical for writing correct JavaScript.
Example 1

Regular Function — this Depends on the Caller

Who calls the function determines what this is

const person = {
  name: 'Vinoth',
  greet: function() {
    console.log('Hi, I am ' + this.name); // this = person
  }
};
person.greet(); // this = person

// Extract and call separately
const fn = person.greet;
try {
  fn(); // this = window/undefined — name is gone!
} catch(e) {
  console.log('Error: ' + e.message);
}
console.log('window.name: ' + (typeof window !== 'undefined' ? window.name || '(empty)' : 'N/A'));
✓ Output
💡 What happened?

When called as person.greet(), this = person. When the same function is extracted into fn and called standalone, this is no longer person — it becomes window (or undefined in strict mode).

Example 2

Arrow Function — this is Lexical

Arrow functions inherit this from where they are WRITTEN

const obj = {
  name: 'Vinoth',

  // Arrow: this comes from enclosing scope (not obj!)
  arrowGreet: () => {
    console.log('Arrow: ' + (typeof this !== 'undefined' ? this.name : 'undefined'));
  },

  // Regular: this = the object that calls it
  regularGreet: function() {
    console.log('Regular: ' + this.name);
  }
};
obj.arrowGreet();    // undefined — this is window
obj.regularGreet();  // 'Vinoth' — this is obj
✓ Output
💡 What happened?

arrowGreet is an arrow function defined inside an object literal. Its this is locked to the surrounding scope (window/global), NOT obj. regularGreet is a regular function — when called as obj.regularGreet(), this = obj.

Example 3

The Classic Problem — setTimeout

"this" gets lost in callbacks; arrow functions fix it

const timer = {
  name: 'Vinoth',

  // Regular fn — this is lost inside setTimeout
  startBad: function() {
    setTimeout(function() {
      console.log('Bad: ' + this.name);  // undefined
    }, 10);
  },

  // Arrow fn — inherits this from startGood
  startGood: function() {
    setTimeout(() => {
      console.log('Good: ' + this.name); // 'Vinoth'
    }, 30);
  }
};
timer.startBad();
timer.startGood();
✓ Output
💡 What happened?

Inside startBad, the regular function callback loses this when setTimeout invokes it. Inside startGood, the arrow function lexically inherits this from startGood (which is timer), so this.name works correctly.

Example 4

Fixing this with call / apply / bind

Manually set what this points to

function greet(city, country) {
  console.log(this.name + ' from ' + city + ', ' + country);
}
const user = { name: 'Vinoth' };

// call — invoke now, args one by one
greet.call(user, 'Minneapolis', 'USA');

// apply — invoke now, args as array
greet.apply(user, ['Minneapolis', 'USA']);

// bind — returns new function, this permanently locked
const bound = greet.bind(user);
bound('Minneapolis', 'USA');
✓ Output
💡 What happened?

All three manually set this to user. call and apply invoke immediately (args differ in format). bind returns a new function with this permanently locked — call it whenever you want.
