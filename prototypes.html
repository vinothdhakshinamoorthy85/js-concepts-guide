Constructor + Prototype Method

Methods on the prototype are shared — one copy for all instances

function Person(name) {

Example 3

Prototype Inheritance — Animal to Dog

Chain constructors together for full inheritance

function Animal(name) { this.name = name; }
Animal.prototype.speak = function() {
  console.log(this.name + ' makes a sound.');
};
function Dog(name, breed) {
  Animal.call(this, name); // inherit properties
  this.breed = breed;
}
// Wire up the prototype chain
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;

const rex = new Dog('Rex', 'Labrador');
rex.speak();
console.log('Breed: ' + rex.breed);
console.log('instanceof Dog? ' + (rex instanceof Dog));
console.log('instanceof Animal? ' + (rex instanceof Animal));
✓ Output
💡 What happened?

Dog.prototype = Object.create(Animal.prototype) wires the chain: rex → Dog.prototype → Animal.prototype → Object.prototype. rex.speak() walks up and finds it on Animal.prototype.

Example 4

Memory: Prototype vs Constructor Method

See the memory difference in practice

// Method inside constructor — new copy per instance (bad)
function PersonBad(name) {
  this.name = name;
  this.sayhi = function() { console.log('Hi ' + this.name); };
}
const b1 = new PersonBad('A'), b2 = new PersonBad('B');
console.log('Bad — same fn? ' + (b1.sayhi === b2.sayhi));

// Method on prototype — shared by all instances (good)
function PersonGood(name) { this.name = name; }
PersonGood.prototype.sayhi = function() { console.log('Hi ' + this.name); };
const g1 = new PersonGood('A'), g2 = new PersonGood('B');
console.log('Good — same fn? ' + (g1.sayhi === g2.sayhi));
✓ Output
💡 What happened?

PersonBad creates a brand-new copy of sayhi for every instance — wastes memory. PersonGood puts sayhi on the prototype so all instances share one copy. For 1000 instances, that's 999 fewer functions in memory.

Prototypes

Object inheritance through a chain of prototype links

Every JavaScript object has a hidden [[Prototype]] link to another object. When you access a property that doesn't exist on the object, JS automatically walks up this prototype chain to find it. The chain ends at Object.prototype, whose prototype is null.
Example 1

Prototype Chain

Every object has a hidden link to another object — its prototype

const animal = {
  breathes: true,
  eat() { console.log('Nom nom...'); }
};
const dog = { barks: true };
Object.setPrototypeOf(dog, animal);

console.log('barks: ' + dog.barks);        // own
console.log('breathes: ' + dog.breathes);  // inherited
dog.eat();                                  // inherited method

console.log('own barks? ' + dog.hasOwnProperty('barks'));
console.log('own breathes? ' + dog.hasOwnProperty('breathes'));
✓ Output
💡 What happened?

dog doesn't have breathes or eat() itself. JS walks up to animal (its prototype) and finds them. hasOwnProperty() confirms that breathes is inherited, not owned.
